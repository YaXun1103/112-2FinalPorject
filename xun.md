使用案例表
### 使用案例表

#### 使用案例1

| 使用案例名稱 | 註冊 |
| ------------- | ---- |
| 使用案例描述 | 讀者使用手機號碼或電子郵件在線上借書系統上註冊成為會員，以便瀏覽圖書館藏書、借閱圖書和使用其他服務。 |
| 主要參與者 | 讀者 |
| 利害關係人與目標 | 1. 讀者：希望能夠快速、簡便地註冊成為會員，以便使用系統的全部功能，包括查詢圖書、借閱圖書和接收還書提醒。<br>2. 圖書館：希望通過會員註冊獲取讀者信息，以便提供更好的服務和管理借書流程。 |
| 前置條件 | 1. 讀者有可用的手機號碼或電子郵件地址。 |
| 後置條件 | 1. 讀者成功註冊成為會員，並可以使用系統的全部功能。<br>2. 系統記錄讀者的基本信息，並向讀者發送歡迎信息或確認郵件。 |
| 主要成功情節 | 1. 讀者打開借書系統並選擇註冊選項。<br>2. 系統提示讀者輸入手機號碼或電子郵件地址。<br>3. 讀者輸入手機號碼或電子郵件地址並提交。<br>4. 系統發送驗證碼到讀者的手機或電子郵件。<br>5. 讀者輸入收到的驗證碼並提交。<br>6. 系統驗證碼驗證成功後，提示讀者設置密碼。<br>7. 讀者設置密碼並提交。<br>8. 系統顯示註冊成功信息，並自動登錄到系統主界面。 |
| 例外情節與其他需求 | 1. 驗證碼發送失敗<br>2. 驗證碼輸入錯誤<br>3. 手機號碼或電子郵件已被使用<br>4. 網絡連接問題<br>5. 安全性需求<br>6. 讀者隱私 |

#### 使用案例2

| 使用案例名稱 | 登錄 |
| ------------- | ---- |
| 使用案例描述 | 讀者使用註冊的手機號碼或電子郵件和密碼登錄到借書系統，以便瀏覽圖書館藏書、借閱圖書和使用其他服務。 |
| 主要參與者 | 讀者 |
| 利害關係人與目標 | 1. 讀者：希望能夠快速、安全地登錄系統，以便使用系統的全部功能。 |
| 前置條件 | 1. 讀者已註冊並有可用的帳號和密碼。 |
| 後置條件 | 1. 讀者成功登錄系統，並可以使用系統的全部功能。 |
| 主要成功情節 | 1. 讀者打開借書系統並選擇登錄選項。<br>2. 系統提示讀者輸入手機號碼或電子郵件和密碼。<br>3. 讀者輸入正確的帳號和密碼並提交。<br>4. 系統驗證成功後，顯示登錄成功信息，並進入系統主界面。 |
| 例外情節與其他需求 | 1. 帳號或密碼輸入錯誤<br>2. 帳號被鎖定<br>3. 網絡連接問題<br>4. 安全性需求<br>5. 讀者隱私 |

#### 使用案例3

| 使用案例名稱 | 查詢圖書 |
| ------------- | ---- |
| 使用案例描述 | 讀者按分類、作者、標題或關鍵字查詢圖書館藏書，並查看圖書的詳細信息。 |
| 主要參與者 | 讀者 |
| 利害關係人與目標 | 1. 讀者：希望能夠快速、準確地查詢到所需的圖書信息，以便決定是否借閱。 |
| 前置條件 | 1. 讀者已登錄系統。 |
| 後置條件 | 1. 讀者看到查詢結果，並可查看圖書的詳細信息。 |
| 主要成功情節 | 1. 讀者在系統中輸入查詢條件（分類、作者、標題或關鍵字）。<br>2. 系統根據查詢條件返回符合的圖書列表。<br>3. 讀者選擇其中一本圖書，查看其詳細信息（書名、作者、出版日期、館藏狀態）。 |
| 例外情節與其他需求 | 1. 查詢條件輸入錯誤<br>2. 系統無法找到符合條件的圖書<br>3. 網絡連接問題<br>4. 安全性需求<br>5. 讀者隱私 |

#### 使用案例4

| 使用案例名稱 | 借書 |
| ------------- | ---- |
| 使用案例描述 | 讀者選擇要借的書籍，加入到借書籃中，確認借書信息後，系統生成借書清單並顯示預計還書日期。 |
| 主要參與者 | 讀者 |
| 利害關係人與目標 | 1. 讀者：希望能夠方便地借閱所需的圖書，並清楚地知道還書日期。 |
| 前置條件 | 1. 讀者已登錄系統。<br>2. 讀者已查詢並選擇了要借的圖書。 |
| 後置條件 | 1. 借書申請成功，系統生成借書清單並顯示預計還書日期。 |
| 主要成功情節 | 1. 讀者在系統中選擇要借的圖書並加入借書籃。<br>2. 系統提示讀者確認借書信息。<br>3. 讀者確認信息並提交借書申請。<br>4. 系統生成借書清單並顯示預計還書日期。 |
| 例外情節與其他需求 | 1. 圖書已被其他讀者借出<br>2. 圖書狀態不允許借閱<br>3. 網絡連接問題<br>4. 安全性需求<br>5. 讀者隱私 |

#### 使用案例6

| 使用案例名稱 | 接收還書提醒 |
| ------------- | ---- |
| 使用案例描述 | 系統在還書日期臨近時自動發送還書提醒通知給讀者。 |
| 主要參與者 | 讀者 |
| 利害關係人與目標 | 1. 讀者：希望能夠在還書日期臨近時收到提醒，避免逾期。<br>2. 圖書館：希望讀者能按時還書，避免圖書逾期。 |
| 前置條件 | 1. 讀者有未還的書籍且還書日期臨近。 |
| 後置條件 | 1. 讀者收到還書提醒通知。 |
| 主要成功情節 | 1. 系統檢查還書日期。<br>2. 系統在還書日期臨近時發送提醒通知給讀者。<br>3. 讀者收到提醒通知。 |
| 例外情節與其他需求 | 1. 系統發送提醒失敗<br>2. 網絡連接問題<br>3. 安全性需求<br>4. 讀者隱私 |
